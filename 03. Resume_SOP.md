# 📄 Resume SOP — Placement Readiness Program

## Standard Operating Procedure
**Resume Preparation — Structure, Guidelines & Do's / Don'ts**

---

## Section A — Resume Structure (Mandatory Order)

| # | Section / Component | Details & Sub-Items |
|---|---------------------|---------------------|
| 1 | **Title** | • Full Name<br>• Email Address<br>• Mobile Number<br>• Designation (Data Science Engineer / Data Analyst / ML Engineer) |
| 2 | **Objectives / Goal / Aim** | Optional — keep concise (1–2 lines). Focus on DS/ML career goal. |
| 3 | **Experience** *(Reverse Chronological)* | • Company Name<br>• Designation<br>• Duration<br>• Roles & Responsibilities (data pipelines, model building, EDA, insights) |
| 4 | **Education** *(Reverse Chronological)* | • Course Name<br>• Course Stream (Computer Science / Statistics / Mathematics preferred)<br>• Completion Year<br>• Academic Project (DS/ML focus) |
| 5 | **Certifications** *(If Any)* | • Certificate Name<br>• Certificate ID (if available)<br>• Validity / Year |
| 6 | **Key Skills / Skill Set** | Add ALL known skills: Python, SQL, R, Machine Learning, Deep Learning, NLP, Pandas, NumPy, Scikit-learn, TensorFlow, PyTorch, Tableau, Power BI, AWS, Azure, GCP, MLflow, Airflow, Spark |
| 7 | **Personal Details** *(Optional)* | • Full Name<br>• Date of Birth<br>• Gender<br>• Permanent Address<br>• Marital Status (add only if unmarried) |

---

## Section B — Resume Do's & Don'ts

| ✅ DO'S | ❌ DON'TS |
|---------|-----------|
| ✓ You can remove the Education section if it is totally irrelevant to the role | ✗ Don't use phrases like "I can", "I will", "I want to", etc. |
| ✓ Use past tense or present continuous tense *(e.g., "Built classification models")* | ✗ Don't explain what technologies are — describe what you did with them |
| ✓ Add ALL key skills — Python, SQL, ML, DL, BI tools, cloud platforms, frameworks | ✗ Don't use client names or project names |
| ✓ Keep Roles & Responsibilities as the core focus | ✗ Don't use words like "know", "learnt", "knowledge", "familiar with" in R&R |
| ✓ Mention certifications with ID and validity wherever available | ✗ Don't use AI-generated content |
| ✓ Keep personal details concise and optional (add marital status only if unmarried) | ✗ Don't divide responsibilities by tool *(e.g., "For Python I did… For SQL I did…")* |

---

## ⚡ Key Reminder

> **Roles & Responsibilities = Core of Resume** · Use **past tense or present continuous** · Add **ALL technical skills** · **No client/project names** · **No "I can/I will" statements**
